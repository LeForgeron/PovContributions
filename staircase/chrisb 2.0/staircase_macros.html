The StairCase Macro

The StairCase macro adds a staircase, or parts of a staircase, to a scene file. Control variables are used to control the path that the staircase follows and the components that are incorporated into the scene file. The following major components are included by default but can be suppressed by setting the appropriate control variable.
Major Component Control Variable
Stairs StairCase_StairsOn Comprised of Risers and Treads.
Main Banister StairCase_MainBanisterOn Comprised of Newel Posts, Balusters, Hand Rail, Toe Rail and Stringer.
Paired Banister StairCase_PairedBanisterOn Comprised of Newel Posts, Balusters, Hand Rail, Toe Rail and Stringer.
Landings StairCase_LandingsOn Comprised of Flooring with optional carpet.
Carpeting StairCase_StairCarpetOn
The StairCase macro can be used to produce a single flight of stairs or a more complex structure with multiple flights of stairs connected together with landings. The path that the staircase follows is based upon the positions of newel posts as specified in the StairCase_PostPositions control variable. The control variables used to control this macro are described with examples in the Variables Reference section of this document.

The StairCase macro is contained within the staircase.inc file along with sufficient default object definitions to generate the default staircase.

All of the control variables have default settings to enable the StairCase macro to be used by POV-Ray users without much experience.

The StairCase macro does not accept any parameters.

The StairCase_Spiral Macro

The StairCase_Spiral macro adds a spiral staircase, or components of a spiral staircase, to a scene file. Control variables control the path that the spiral staircase will follow and the components to be rendered. The following major components are included by default but can be suppressed by setting the appropriate control variable.
Major Component Control Variable
Stairs StairCase_StairsOn Comprised of Risers and Treads.
Main Banister StairCase_MainBanisterOn Comprised of Newel Posts, Balusters, Hand Rail, Toe Rail and Stringer.
Paired Banister StairCase_PairedBanisterOn Comprised of Newel Posts, Balusters, Hand Rail, Toe Rail and Stringer. The paired banister is suppressed by default.
Carpeting StairCase_StairCarpetOn
Spiral Core StairCase_SpiralCoreOn The spiral core is a cylindrical column at the centre of the staircase.
The SpiralStairCase macro can be used to produce a single spiral flight of stairs. The path that the staircase follows is based upon the positions of newel posts as specified in the StairCase_PostPositions control variable. The control variables used to control this macro are described with examples in the Variables Reference section of this document.

The SpiralStairCase macro is contained within the staircase.inc file along with sufficient default object definitions to generate the default spiral staircase which consists of a spiral centred at the origin and rising up around the +y axis. The default stair width is 1 POV-Ray unit which is intended to represent 1 metre.

All of the control variables have default settings to enable the SpiralStairCase macro to be used by POV-Ray users without much experience.

The SpiralStairCase macro does not accept any parameters.

The StairCase_Undef Macro

This macro resets control variables by issuing an '#undef' for any that have been defined. This essentially re-initialises settings so that they revert to their default values the next time one of the staircase macros is called.

For example, the sample scene file 'staircase_example05.pov' creates a set of stone steps with a stonework balustrade around a building. It then calls the 'StairCase_Undef' macro to revert to the default wooden staircase components and textures before generating the interior staircases and perimeter handrails.

The StairCase_BannisterGen Macro

This macro is used by the StairCase macro to define the banister rail that connects two newel posts. Certain variables must be set before this macro is called. This macro is not intended to be called directly from a scene file. This macro requires 3 parameters:
Start A 3D vector defining the position of the base of the newel post at one end of the rail.
End A 3D vector defining the position of the base of the newel post at the other end of the rail.
Height The height, in POV-Ray units of the handrail.

The StairCase_SpiralBannisterGen Macro

This macro is used by the StairCase_Spiral macro to define the banister rail that connects two newel posts on a spiral staircase. Certain variables must be set before this macro is called. This macro is not intended to be called directly from a scene file. This macro requires 5 parameters:
Start A 3D vector defining the position of the base of the newel post at one end of the rail.
Height The height of the spiral, which is usually the same as the distance from the top of the first step to the top of the last step.
PositionDelta A vector indicating the amount that each slice of the spiral needs to be moved out (or in) by.
StartRadius The radius of the spiral at the start of the handrail.
EndRadius The radius of the spiral at the end of the handrail

The StairCase_Compatibility Macro

This macro is called by the StairCase macro if the StairCase_CompatibilityMode variable is set to '1', to provide a degree of compatibility with the old POV-Stairs package. The primary difference between the two is that POV-Stairs used variable names with a prefix of 'sc' where this version uses the prefix 'StairCase_' (in order to conform to the naming conventions of the POV-Ray Object Collection). For any 'current' variable that is not set, where the 'old' variable name is set, this macro copies the value across to the corresponding current variable names.

Note This is not a foolproof technique where you call the 'StairCase' macro more than once in a single scene file because the first call to the macro will initialise all variables, so any settings assigned to old variable names between an initial call to the 'StairCase' macro and a second call to the macro will be ignored. The recommended approach is to go through your scene file and rename variables. This can usually be achieved pretty quickly using the search and replace option provided by most text editors (including the POV-Ray Windows editor).

The StairCase_NewelShape Macro

This macro is defined in the file 'StairCase_Samples.inc' and can be used to define a newel post based upon a set of standard definitions embedded within the macro. You can use this macro as illustrated in the description for the use of the StairCase_NewelPost variable.

The StairCase_BalusterShape Macro

This macro is defined in the file 'StairCase_Samples.inc' and can be used to define a baluster based upon a set of standard definitions embedded within the macro. You can use this macro as illustrated in the description for the use of the StairCase_Baluster variable.
