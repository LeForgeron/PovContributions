Main Control Variables

2.1.1  StairCase_PostPositions

The StairCase_PostPositions array holds a sequence of coordinates defined as 3D vectors. Each coordinate vector defines the position of a newel post and the sequence of positions therefore defines the path that will be followed by one side of the staircase. This is therefore the most important control variable for POV-Sairs as it effectively defines the path of the entire staircase.

The StairCase macro adds newel posts at the positions specified in the StairCase_PostPositions array and optionally adds corresponding newel posts on the opposite side of the staircase. It also optionally adds stairs, landings and banister rails to connect the newel posts together.

The StairCase_Spiral macro only uses the first two elements of this array. It adds newel posts at those two positions and calculates a spiral that passes through those positions and is centred on a line passing vertically through the point defined by the StairCase_SpiralCentre control variable. This macro also optionally adds stairs, landings and banister rails to connect the newel posts together.

It is anticipated that, other than for simple testing purposes you will always override the default values assigned to this array. This is a standard POV-Ray array and as such can be declared in a number of different ways as detailed in the POV-Ray documentation. Two common ways of declaring this variable are shown below. The first declares the array and assigns its values in a single statement.

#declare StairCase_PostPositions = array [6] {<-3,0,0>,<-2,0,0>,<-1,-1,0>,<1,-1,0>,<2,0,0>,<3,0,0>};
  • array [6] tells POV-Ray that this variable is an array with 6 elements.
  • 6 sets of 3D <x,y,z> coordinates are then specified within the curly brackets and separated by commas.
  • The variable declaration is terminated with a semicolon (';').

The following example declares and dimensions the array identifier in the first statement, then assigns its 6 values in separate statements.

#declare StairCase_PostPositions = array [6];
#declare StairCase_PostPositions[0] = <-3, 0,0>;
#declare StairCase_PostPositions[1] = <-2, 0,0>;
#declare StairCase_PostPositions[2] = <-1,-1,0>;
#declare StairCase_PostPositions[3] = < 1,-1,0>;
#declare StairCase_PostPositions[4] = < 2, 0,0>;
#declare StairCase_PostPositions[5] = < 3, 0,0>;
  • A 6 element array is declared.
  • A set of 3D <x,y,z> coordinates is assigned to each array element, starting from element 0 and ending at element 5.
  • Each declaration is terminated with a semicolon (';').
Note that all of the StairCase_PostPositions array elements needs to be assigned a valid 3D vector, otherwise the StairCase macro will return an error.

Numerous examples illustrating the use of this control variable are included in the quick start section of this documentation.

2.1.2  StairCase_StairWidth

This variable is used to define the width of the risers and treads (the steps of the stair case). If the StairCase_StairHandedness control variable is set to '1' or '-1' then the StairCase_StairWidth control variable also defines the centre to centre distance from one banister rail to the opposite banister rail. However, the StairCase_StairHandedness can also be set to a non whole number, in which case the risers and treads can protrude beyond the line of the banister rails. By default it is set to '1' (1 metre). To override the default setting you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairWidth = 4;
StairCase()
This example sets the stair width to 4 units.

2.1.3  StairCase_StairCarpetWidth

This variable is used to define the width of the carpet. By default it is set to be the same as the stair width as defined using the StairCase_StairWidth control variable. Reducing this width leaves an uncarpeted gap at the edges of the stairs. To override the default setting you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairCarpetWidth = 0.5;
StairCase()
This example sets the stair carpet width to 0.5 POV-Ray units. As shown in the image on the right, the stair width (as defined using the StairCase_StairWidth variable) has defaulted to a value of 1, making the stairs 1 POV-Ray unit wide. The carpet does not therefore cover the whole stair width, leaving bare sections of riser and tread on either side of the carpet.

2.1.4  StairCase_StairHandedness

This variable is used to tell the StairCase macro whether the left hand or the right hand newel post positions have been specified in the StairCase_PostPositions array. If set to '1' then these newel posts will be on the left of the staircase and if set to '-1' the newel posts will be to the right of the staircase. A non-whole number can be specified to move the posts and the banister rails in from the end of the stairs. By default this value is set to '1'. To override the default setting you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairHandedness = -1;
#declare StairCase_PostPositions = array [3] {<1,0,0>,<1,1,1.25>,<2.25,2,1.25>};
StairCase()
Note that it is your responsibility to define newel post positions, a stair width and a stair handedness that provide enough space for stairs to be constructed without overlapping. If you StairCase_StairHandedness were to be assigned a value of '1' with the post positions that were used in the exampel above, then the right turn made by the handrail would cause the start of the second flight of stairs to overlap the end of the first flight of stairs. The StairCase macro will not report this as an error, but the staircase you end up with will look a mess.

2.1.5  StairCase_HandRailHeight

This variable is used to define the height of the hand rail above the floor level of the newel posts. By default it is set to '0.920' (92cm). To override the default setting you can declare this variable in your scene file before calling the StairCase macro. For example:

#include "staircase.inc"
#declare StairCase_HandRailHeight = 0.925;
StairCase()

2.1.6  StairCase_MaxBalusterSpacing

This variable is used to define the maximum spacing for the balusters. The smallest number of balusters that will span the gap between two Newel Posts without leaving a gap larger than this value will be generated. By default it is set to '0.135' (13.5cm). To override the default setting and specify a maximum you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_MaxBalusterSpacing = 0.008;
StairCase()
Note that there is nothing stopping you specifying a very small baluster spacing that will cause the balusters to overlap.

2.1.7  StairCase_MaxRiserHeight

This variable is used to define the maximum height of a riser. The smallest number of risers that will rise the height between two Newel Posts without an individual riser being higher than this value will be generated. By default it is set to '0.190' (19cm). To override the default setting and specify a maximum riser height you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_MaxRiserHeight = 0.10;
StairCase()
The example illustrated in the image to the right specifies a very small riser height, creating a flight of stairs that is not particularly practical.
You can also specify the proportion of the riser that will be included in the staircase, to give an open effect. See StairCase_RiserCoverage

2.1.8  StairCase_LandingThreshold

This variable is used to define an angle that serves as a threshold above which an extra Newel Post will be added by the StairCase macro when creating a landing. By default it is set to '50' (50 degrees). If successive sections of staircase turn by more than this amount, an extra Newel Post will be added to assist with the turn between flights of stairs. To override the default setting you can declare this variable in your scene file before calling the StairCase macro. For example:

#include "staircase.inc"
#declare StairCase_LandingThreshold = 30;
#declare StairCase_PostPositions = array [3] {<0,0,0>,<0,1,1.25>,<-0.75,2,2.5>};
StairCase()
When the post positions create a landing that turns through an angle that exceeds the threshold an extra newel post is added.
#include "staircase.inc"
#declare StairCase_LandingThreshold = 40;
#declare StairCase_PostPositions = array [3] {<0,0,0>,<0,1,1.25>,<-0.75,2,2.5>};
StairCase()
When the post positions create a landing that turns through an angle that is less than the threshold the last newel post from one flight of stairs is connected to the first newel post from the next flight.

2.1.9  StairCase_RiserCoverage

This variable is used to define the proportion of the riser that is present. If set to 1 then the whole riser will be present giving a staircase with no open gaps between the treads. A value of 0.5 will fill the upper half of the riser leaving a gap between the base of each riser and the top of the next tread down. By default this value is set to '1'. To override the default setting you can declare this variable in your scene file before calling the StairCase macro. For example:

#include "staircase.inc"
#declare StairCase_RiserCoverage = 0.5;
#declare StairCase_StairCarpetOn = 0;
StairCase()
This tells the StairCase Macros to only use half height risers to support the treads, giving an open style of staircase. The rise from one tread to the next is controlled using the StairCase_MaxRiserHeight control variable.

2.1.10  StairCase_MaxInclination (StairCase_Spiral Macro only)

This variable is used to define the maximum inclination of the outer edge of a spiral staircase. This would typically be set to '3/4' (=0.75) or less so that the spiral staircase does not get too steep. The StairCase_Spiral macro uses this value to work out whether to add extra 360 degree turns to the staircase in order to raise it through the height necessary to join the starting point to the end point as defined using the first two elements of the StairCase_Positions array. The default setting for this variable is '3/4' representing a rise of 3 units for every 4 horizontal units. To override the default setting you can declare this variable in your scene file before calling the StairCase_Spiral macro.

The following examples all use the same start and end, x and z positions the height (the y dimension) of the end point is changed for the second and third example:

This first example shows a stretch of staircase that comfortably rises 2.35 metres without breaching the default limit for StairCase_MaxInclination of '3/4'. Increasing the height to 3 metres exceeds a rise of 3 in 4 at the outer edge, so an extra loop is automatically added into the staircase. Increasing the maximum permitted inclination to '39/40' gives an almost 45 degree inclination around the outer edge, resulting in very steep stairs, but avoiding the extra 360 degree loop in the spiral.
#include "staircase.inc"
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<0,2.35,2>};
StairCase_Spiral ()

#include "staircase.inc"
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<0,3,2>};
StairCase_Spiral ()

#include "staircase.inc"
#declare StairCase_MaxInclination = 39/40;
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<0,3,2>};
StairCase_Spiral ()

2.1.12  StairCase_CentralCoreRadius (StairCase_Spiral Macro only)

This variable is used to define the radius of the central cylindrical core of a spiral staircase. This value is used even if the StairCase_CentralCoreOn control variable is set to '0'. The default setting for this variable is '0.1' which results in a vertical cylinder with a diameter 0.2 units being added to the spiral staircase. To override the default setting you can declare this variable in your scene file before calling the StairCase_Spiral macro. For example:

This first example uses a spiral staircase with a lower newel post at <-2,0,0>, an upper newel post at <2,2,0> and a central core with the default radius of 0.1. When the radius is set to 0.5 the staircase wraps itself around a wider central core.
#include "staircase.inc"
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<2,2.5,0>}; StairCase_Spiral ()

#include "staircase.inc"
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<2,2.5,0>}; StairCase_Spiral ()

#include "staircase.inc"
#declare StairCase_CentralCoreRadius = 0.2;
StairCase_Spiral()
This tells the StairCase Macros to permit a define a cylindrical core of 0.2 units in radius.

Control Switches

2.2.1  StairCase_StairsOn

This variable is used to indicate whether the StairCase macro should define stairs or just the banister rails. By default it is set to '1' to define a complete stair case. To override the default setting you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairsOn = 0;
StairCase()
This example suppresses the generation of stairs as shown in the image on the right. Notice that this also suppresses the generation of landings, which, for the purposes of this setting are considered to be part of the stair assembly.

2.2.2  StairCase_LandingsOn

This variable is used to indicate whether the StairCase macro should define landings to join the various flight of stairs together. By default it is set to '1' to define a complete stair case. To override the default setting you can declare this variable in your scene file before calling the StairCase macro. For example:

#include "staircase.inc"
#declare StairCase_LandingsOn = 0;
StairCase()
This example suppresses the generation of landings as shown in the image on the right.

2.2.3  StairCase_StairCarpetOn

This variable is used to indicate whether the StairCase macro should carpet the stairs. By default it is set to '1' to define a complete stair case. To override the default setting you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairCarpetOn = 0;
StairCase()
This example suppresses the stair carpet as shown in the image on the right.

2.2.4  StairCase_MainBanisterOn

This variable is used to indicate whether the StairCase macro should include the main banisters. This is the banister whose position is defined by the positions of the Newel Posts specified using the StairCase_PostPositions array. By default this variable is set to '1' to define a stair case with a banister rail. If set to zero, the Newel Posts whose positions are specified are not included in the generated staircase. This may seem unusual, but can be useful when constructing a staircase against a wall, because it may be easier to specify the positions against the wall than to calculate the positions of the corresponding banisters on the opposite side of the staircase. To override the default setting you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_MainBanisterOn = 0;
StairCase()
This example suppresses the main banister as shown in the image on the right.
Note that the banister rail referred to as the main banister rail is dependant upon the handedness of the stairs. In this example the setting for the StairCase_StairHandedness variable defaults to '1' which indicates that the stairs are drawn to the right of the specified newel posts as one looks at the first flight of stairs from the starting point.

2.2.5  StairCase_PairedBanisterOn

This variable is used to indicate whether the StairCase macro should include the 'paired' banisters. That is to say the banisters on the opposite side of the staircase to the banisters whose positions are defined by the positions of the Newel Posts specified using the StairCase_PostPositions array. By default, the StairCase macro uses a value of '1' for this variable, whereas the StairCase_Spiral macro uses a value of '0' for this variable. To override the default setting you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_PairedBanisterOn = 0;
StairCase()
This example suppresses the paired banister as shown in the image on the right.
Note that the banister rail referred to as the paired banister rail is dependant upon the handedness of the stairs. In this example the setting for the StairCase_StairHandedness variable defaults to '1' which indicates that the stairs are drawn to the right of the specified newel posts as one looks at the first flight of stairs from the starting point. Setting StairCase_PairedBanisterOn to '0' therefore suppresses the banister rail to the right of those stairs.

2.2.6  StairCase_CentralCoreOn (StairCase_Spiral Macro only)

This variable is used to define whether a central cylindrical core is added to the spiral staircase. The StairCase_CentralCoreRadius control variable is used to set the radius of the core. The default setting for this variable is '1' which results in a vertical cylindrical core being added to the spiral staircase. To override the default setting you can declare this variable in your scene file before calling the StairCase_Spiral macro.

This first example uses a visible 0.5 unit radius central core. This second example switches the visible cylindrical core off.
#include "staircase.inc"
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<2,2.5,0>};
#declare StairCase_CentralCoreRadius = 0.5;
StairCase_Spiral ()

#include "staircase.inc"
#declare StairCase_CentralCoreOn =0;
#declare StairCase_CentralCoreRadius = 0.5;
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<2,2.5,0>};
StairCase_Spiral ()

2.2.7  StairCase_ConstantStairWidthOn (StairCase_Spiral Macro only)

The StairCase_Spiral macro can create a spiral where the inner edge of the spiral follows the edge of a cylinder of a radius specified using the StairCase_CentralCoreRadius control variable and centred at the point defined usind the StairCase_SpiralCentre control variable, or it can maintain a constant stair width. If the starting point and the ending point are an equal distance from the centre line of the spiral, then there is little evident difference.

The default setting for this variable is '0' which causes the inner edge of the staircase to follow a cylindrical spiral. To override the default setting you can declare this variable in your scene file before calling the StairCase_Spiral macro. In the following example the newel posts are very different distances from the centre of the spiral (which defaults to <0,0,0>) so that the effect of this setting becomes more apparent.

This first example sets 'StairCase_ConstantStairWidthOn' to '0' (the default). The inner edge of the spiral therefore follows the cylindrical central core which has been given a radius of 0.5 to show this effect more clearly. This second example sets it to '1', indicating that the spiral of the inner edge should remain parallel to the spiral of the outer edge.
#include "staircase.inc"
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<2,2.5,4>};
#declare StairCase_CentralCoreRadius = 0.5;
#declare StairCase_ConstantStairWidthOn = 0;
StairCase_Spiral ()

#include "staircase.inc"
#declare StairCase_PostPositions = array [2] {<-2,0,0>,<2,2.5,4>};
#declare StairCase_CentralCoreRadius = 0.5;
#declare StairCase_ConstantStairWidthOn = 1;
StairCase_Spiral ()

2.2.8  StairCase_CompatibilityMode (StairCase Macro only)

If this variable is set to '1' the StairCase macro calls the StairCase_Compatibility macro to provide a degree of compatibility with the old 'POV-Stairs' package. The primary difference between the two is that POV-Stairs used variable names with a prefix of 'sc' where this version uses the prefix 'StairCase_'. For any 'current' variable that is not set, where the 'old' variable name is set, this macro copies the value across to the corresponding current variable names.

Note This is not a foolproof technique where you call the 'StairCase' macro more than once in a single scene file because the first call to the macro will initialise all variables, so any settings assigned to old variable names between an initial call to the 'StairCase' macro and a second call to the macro will be ignored. The recommended approach is to go through your scene file and rename variables. This can usually be achieved pretty quickly using the search and replace option provided by most text editors (including the POV-Ray Windows editor).

Object Definitions

2.3.1  StairCase_NewelPost

This object represents a Newel Post. A pair of Newel Posts are used to support the ends of a length of handrail. The Newel Post is defined as being about 1 POV-Ray unit in height. The edge that will end up pointing towards the start of the flight of stairs needs to be initially defined pointing in the +z direction. The StairCase macro applies a texture based upon the setting of the StairCase_NewelTexture variable. It then rotates and translates each Newel Post into position as necessary.

If not set, this object will default to the shape shown on the left which consists of a lathe cut shape. To override the default setting you can declare this object in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_NewelPost = intersection {
   box {<-0.042,-1,-0.042><0.042,1.02,0.042>}
   plane {<0,1,1>,0 translate 1.05*y}
   plane {<0,1,1>,0 translate 1.05*y rotate 90*y}
   plane {<0,1,1>,0 translate 1.05*y rotate 180*y}
   plane {<0,1,1>,0 translate 1.05*y rotate 270*y}
}
StairCase()
This example defines a simple box shape for the post then bevels off the four top edges by intersecting the post with four appropriately positioned planes.

Box - Bevelled - Chamfer - Notched - Turned - Fluted 		A number of sample newel post shapes are included in the 'StairCase_Samples.inc' include file. The definitions are stored inside the StairCase_NewelShape macro, so to use these shapes you need to include the file and run the macro, assigning the result to the 'StairCase_NewelPost' variable before calling the StairCase or the StairCase_Spiral macro. The following example illustrates this:
#include "StairCase_Samples.inc"
#include "staircase.inc"
#declare StairCase_NewelPost = StairCase_NewelShape ("Fluted")
StairCase()
This example would generate a staircase using the 'Notched' newel post shape shown in the image to the left.

2.3.2  StairCase_Baluster

This object represents a Baluster. A set of Balusters are used to support the handrail between pairs of Newel Posts. It is defined as being a little over 1 POV-Ray unit in height. The StairCase macro trims the top and bottom to match the setting of the StairCase_HandRailHeight variable. The edge that will end up pointing towards the start of the flight of stairs is defined pointing towards the +z direction. The StairCase macro will apply a texture based upon the setting of the StairCase_BalusterTexture variable. It then rotates and translates each Baluster into position as necessary.

If not set, this object will default to the shape shown on the left which consists of a lathe cut shape. To override the default setting you can declare this object in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_Baluster = box {<-0.015,-0.03,-0.015><0.015,1,0.015>};
StairCase()
This gives a simple square section baluster as shown in the image to the right.

Fluted - Turned - Notched - Chamfer - Bevelled - Box 		A number of sample baluster shapes are included in the 'StairCase_Samples.inc' include file. The definitions are stored inside the StairCase_BalusterShape macro, so to use these shapes you need to include the file and run the macro, assigning the result to the 'StairCase_Baluster' variable before calling the StairCase or the StairCase_Spiral macro. The following example illustrates this:
#include "StairCase_Samples.inc"
#include "staircase.inc"
#declare StairCase_Baluster = StairCase_BalusterShape ("Notched")

This example would generate a staircase using the 'Notched' baluster shape shown in the image to the left.

2.3.3  StairCase_HandRail

This object represents the hand rail and is defined as being of a 1 POV-Ray unit in length. The StairCase macro will scale this length as necessary to enable the hand rail to span the distance between two Newel Posts. The hand rail is defined starting at the origin and running out along the +z axis. The StairCase macro will scale this section, apply a unique texture based upon the setting of the StairCase_RailTexture variable with a degree of randomness designed to give each hand rail a unique texture. It then rotates and translates each hand rail into position as necessary.

If not set, this object will default to the shape shown on the left. To override the default setting you can declare this object in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_HandRail = cylinder {<0,0.015,0><0,0.015,1>,0.03 clipped_by{plane{-y,0}} scale <1,0.6,1>};
StairCase()
This gives a rounded handrail with a flat underside as shown in the image to the right.

2.3.4  StairCase_Stringer

This object represents the side edging of the stairs. The risers and the treads are connected into this component which supports the weight of a flight of stairs. If not set, this object will default to the shape shown on the left which consists of a simple rectangular piece topped with a standard shoerail. To override the default setting you can declare this object in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:
#include "staircase.inc"
#declare StairCase_Stringer = box {<-0.03,-0.14,-0.08><0.03,0.12,1.08>};
StairCase()
This uses a thicker stringer with no Shoerail, as shown in the image to the right.

2.3.5  StairCase_Tread

This object represents the tread and is defined as being of a 1 POV-Ray unit width. The StairCase macro will scale this width to match the setting of the StairCase_StairWidth variable. The tread is defined at the origin with its top surface at a height of '0' units. The front edge of the tread points towards the +z direction. The back edge is typically defined as being at 1 POV-Ray back in the -z direction. The StairCase macro will trim this so that it stops when it meets the next riser. The StairCase macro will apply a unique texture based upon the setting of the StairCase_TreadTexture variable with a degree of randomness designed to give each stair a unique texture. It then rotates and translates each tread into position as necessary.

If not set, this object will default to the shape shown on the left which consists of a rectangular piece fronted with a rounded edge. To override the default setting you can declare this object in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_Tread = box {<-0.5,-0.042,-1><0.5,0,0.15>};
#declare StairCase_StairCarpetOn = 0;
StairCase()
This replaces the rounded tread with a thick straight edged plank as shown in the image to the right.

2.3.6  StairCase_Riser

This object represents the riser that joins the back edge of one stair to the underside of the next. It is defined as being of a 1 POV-Ray unit in width and nearly 1 POV-Ray unit in height. It's top surface stops just short of y=0 to avoid any problems with coincident surfaces. The StairCase macro will scale the riser width to match the setting of the StairCase_StairWidth variable and will scale the height to match the riser height which is calculated based upon the setting of the StairCase_MaxRiserHeight variable. The riser is defined at the origin with its front surface pointing in the +z direction. The StairCase macro will apply a unique texture based upon the setting of the StairCase_RiserTexture variable with a degree of randomness designed to give each stair a unique texture. It then rotates and translates each tread into position as necessary.

If not set, this object will default to the shape shown on the left which consists of a simple rectangular box. To override the default setting you can declare this object in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_Riser = difference {
box {<-0.5,-1,0.120><0.5,-0.001,0.128>}
box {<-0.05,-0.05,0.119><0.05,0.05,0.129> rotate z*45 scale <1,2,1> translate <-0.25,-0.55,0>}
box {<-0.05,-0.05,0.119><0.05,0.05,0.129> rotate z*45 scale <1,2,1> translate < 0 ,-0.55,0>}
box {<-0.05,-0.05,0.119><0.05,0.05,0.129> rotate z*45 scale <1,2,1> translate < 0.25,-0.55,0>}
}
#declare StairCase_StairCarpetOn = 0;
StairCase()
This example uses CSG to create a slightly more ornate riser as shown in the image to the right.

2.3.7  StairCase_StairCarpet

This object represents a section of carpet running along the top of a tread and down over a riser. It is defined as being of a 1 POV-Ray unit in width and the StairCase macro scales this width to match the setting of the StairCase_StairCarpetWidth variable. The front edge of the carpet segment points towards the +z direction. The back edge is typically defined as being at 1 POV-Ray back in the -z direction. The StairCase macro will trim this so that it stops when it meets the next riser. The StairCase macro will apply a texture based upon the setting of the StairCase_StairCarpetTexture variable. It then rotates and translates each stair carpet section into position as necessary.

If not set, this object will default to the shape shown on the left which consists of a curled shape that wraps around the front of a stair. To override the default setting you can declare this object in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairCarpet = box {<-0.4,0,-0.1><0.4,0.001,0.10>}
StairCase()
This represents a rectangular section of carpet embedded into the tread as shown in the image to the right.

Textures

2.4.1  StairCase_NewelTexture

This variable is used to define a texture for the Newel Posts. By default it is set to T_Wood1 from the "woods.inc" include file supplied with POV-ray. To override the default setting and specify a texture for a Newel Post you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_NewelTexture = texture {pigment {color rgb <1,0,0>}};
StairCase()
As can be seen in this image, the texture specified for the Newel Post also serves as the default texture for other components such as the Balusters and the Hand Rail.

2.4.7  StairCase_StairCarpetTexture

This variable is used to define a texture for the Carpet. By default it is set to have a pigment as defined using the StairCase_StairCarpetColour control variable and an 'agate' normal scaled to provide a disrupted surface normal. To override the default setting and specify a texture for the carpet you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairCarpetTexture = texture {pigment {color rgb <1,1,1>}};
#declare StairCase_StairCarpetWidth = 0.75;
StairCase()

2.4.8  StairCase_StairCarpetColour

This variable is used to define a colour for the Carpet. By default it is set to '<1,1,0>' (Yellow). To override the default setting and specify a texture for the Stair Carpet you can declare this variable in your scene file before calling the StairCase or the StairCase_Spiral macro. For example:

#include "staircase.inc"
#declare StairCase_StairCarpetColour = <0,1,0>;
StairCase()
Note. If you specify a setting for the StairCase_StairCarpetTexture control variable, then the StairCase_StairCarpetColour control variable is ignored.
